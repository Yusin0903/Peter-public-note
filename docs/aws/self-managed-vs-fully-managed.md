# 自己管 vs 全託管（Self-managed vs Fully managed）

## 自己管（MySQL on EC2）

你要自己處理所有事情：

```
你負責：
├── 開 EC2 裝 MySQL
├── 設定 CPU / Memory
├── 硬碟快滿了要擴容
├── 定期備份
├── 版本升級 / 安全性修補
├── 主從複製（高可用）
├── 監控、告警
└── 壞了要自己修
```

## 全託管（DynamoDB）

AWS 幫你處理，你只管用：

```
AWS 負責：
├── 硬體、OS、資料庫引擎
├── 自動擴縮容量
├── 自動備份
├── 自動跨 AZ 複製（高可用）
├── 安全性修補
└── 監控

你只需要：
├── 建表
├── 讀寫資料
└── 付錢（按用量計費）
```

## 一句話

- 自己管 = 你買食材自己煮
- 全託管 = 你去餐廳點餐，廚房的事不用管
